# Internationalization (i18n)

> **Framework**: i18next + react-i18next
> **Languages**: English (default), Chinese (zh-CN)
> **Auto-detection**: localStorage → navigator language

---

## Overview

MaxLaunchpad uses [i18next](https://www.i18next.com/) with [react-i18next](https://react.i18next.com/) for renderer internationalization. React UI strings are externalized into JSON translation files, enabling easy addition of new languages without modifying component code.

## File Structure

```
src/renderer/i18n/
├── index.ts        # i18n initialization & configuration
├── en.json         # English translations
└── zh-CN.json      # Chinese translations
```

## Translation Keys

Keys are organized under these top-level namespaces:

| Namespace | Covers | Example Keys |
|-----------|--------|--------------|
| `app` | App-level renderer states | `app.loading` |
| `menu` | TopBar menu items, tooltips, search placeholder | `menu.file`, `menu.view`, `menu.searchPlaceholder` |
| `contextMenu` | Renderer context menu actions | `contextMenu.edit`, `contextMenu.openFileLocation` |
| `errors` | Renderer-generated error dialog titles/messages | `errors.launchFailedTitle`, `errors.failedToSaveConfiguration` |
| `modals` | Modal dialogs (EditKey, Options, Hotkey, About) | `modals.editKey.label`, `modals.options.title`, `modals.common.save` |

## Integration Guide

### 1. Install Dependencies

```bash
npm install i18next react-i18next i18next-browser-languagedetector
```

### 2. Import i18n Configuration

In `src/renderer/index.tsx`, add before the App component:

```tsx
import './i18n';
```

### 3. Use in Components

```tsx
import { useTranslation } from 'react-i18next';

function MyComponent() {
  const { t } = useTranslation();
  return <div>{t('menu.file')}</div>;
}
```

### 4. Add Language Selector

In `OptionsModal.tsx`:

```tsx
import { useTranslation } from 'react-i18next';

function OptionsModal() {
  const { i18n, t } = useTranslation();
  // ...
  return (
    <select
      value={i18n.language}
      onChange={(e) => i18n.changeLanguage(e.target.value)}
    >
      <option value="en">English</option>
      <option value="zh-CN">中文</option>
    </select>
  );
}
```

## Adding a New Language

1. Create `src/renderer/i18n/<lang>.json` by copying `en.json` and translating all values
2. Register in `src/renderer/i18n/index.ts`:

```typescript
import ja from './ja.json';

const resources = {
  en: { translation: en },
  'zh-CN': { translation: zhCN },
  ja: { translation: ja },
};
```

3. Add the language option in the `OptionsModal` selector

## Language Detection

The detection order is:
1. `localStorage` (persisted user preference)
2. `navigator.language` (browser/system language)

If the detected language is not available, it falls back to `en`.

## Notes

- User data (labels, descriptions, tab names) is **not** affected by i18n — users can enter text in any language regardless of UI locale
- Electron main-process UI, such as tray menus and native file dialog titles, is not part of the renderer i18n layer yet
- The font stack already includes `PingFang SC` for CJK rendering
- All file I/O uses UTF-8 encoding
